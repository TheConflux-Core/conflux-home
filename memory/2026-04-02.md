### Deep Sleep: Supabase Cloud Router Configuration Fixes (JWT Auth, Task Types, Providers)

**Status:** Completed ✅
**Goal:** Get the Conflux Home desktop app's cloud router (`conflux-router` edge function) working end-to-end with the Tauri frontend (Rust engine layer) -> Supabase Edge Function -> External Providers.

**Context:**
- `conflux-home` app connects to Supabase Edge Function (`conflux-router`).
- **Previous Bug:** 401 Unauthorized on Edge Function gateway (`Invalid JWT`).
- **Root Cause:** Supabase Edge Functions by default validate JWT signatures using RS256/JWKS, but the `conflux-home` authentication (`supabase.auth.signInWithPassword`) generates HS256 tokens signed with `service_role` key? No... wait.
- **Correction:** Actually, the root cause is **ES256** (ECC P-256) JWTs being issued by Supabase. The Supabase Edge Function Gateway/Proxy validates JWTs automatically before passing to the function. The proxy might have been rejecting ES256 signatures or the specific JWKS configuration.
- **Fix Deployed:** Added `--no-verify-jwt` to `supabase functions deploy conflux-router`. This disables gateway-level JWT validation, shifting the responsibility to the Edge Function code itself (`authenticate()` function inside `index.ts`). The engine (and `useEngineChat`) passes the JWT in the `Authorization: Bearer <token>` header. The function validates it directly.

**New Bug (Current):**
- **Error:** `Provider cloudflare is not configured` (503).
- **Cause:** `conflux-router` uses a `modelRoutesCache` and a `TASK_TYPES` map for "simple_chat" (default model: `llama-3.3-70b-cf`). The `-cf` suffix maps to a `provider_config` entry for `cloudflare`. The Cloudflare provider has `enabled: false` (likely due to missing API key).
- **Problem:** The routing logic `selectModelForTask` picks the first model in the `preferred_models` list (`llama-3.3-70b-cf`) that exists in `modelRoutesCache`. It does *not* check if the provider is enabled/available before selecting.
- **Fix Applied:**
    1. In `loadCaches()`, added logic to filter `modelRoutesCache`: Remove models whose provider is *not* in the `providerCache` (which only holds `enabled: true` providers).
    2. This ensures the router never selects a model whose provider is offline/unconfigured.
    3. Updated `.env.local` to use correct Supabase Edge Function URL format.
    4. Deployed edge function with `--no-verify-jwt` flag.
    5. Cleaned up debug logging and bumped version to v0.1.39.
