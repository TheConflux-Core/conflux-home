## Compilation Issues Fixed

### Option<&str> in params![]
- `add_reminder`, `add_document`, `add_knowledge` all failed because `Option<&str>` doesn't implement `Display` in `params![]`
- Fix: convert Option values to owned Strings first (`mid.map(String::from)`) before passing to params

### params_from_iter not available in rusqlite 0.31
- Dynamic filter queries (`get_documents`, `get_knowledge`) used `params_from_iter` which doesn't exist in 0.31
- Fix: use match arms for all filter combinations instead of building dynamic SQL with vec params

### get_life_dashboard outside impl block
- The function was accidentally placed after the closing `}` of `impl EngineDb`
- Fix: removed extra `}` and moved function inside the impl block
