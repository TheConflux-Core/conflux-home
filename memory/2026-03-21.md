### Key Technical Solutions
- **Pipeline state sync**: System cron + bash script → GitHub → Vercel API route
- Avoided SSH to cloud server (password auth disabled) — used GitHub as bridge instead
- **R3F vs CSS**: CSS + requestAnimationFrame cleaner for orbital animation than Three.js
- **Vercel deployment**: Cron snapshot commits can interfere with deploys if timing overlaps
- **z-index stacking**: Moving orbs need dynamic z-index on hover to avoid popup occlusion

### Lessons Learned
- R3F `torusGeometry` orbits and JS-positioned orbs need identical math (no z-compression mismatch)
- `<line>` conflicts with SVG in R3F — use imperative `THREE.Line` instead
- Vercel CLI token (`vck_`) != API bearer token — need `vca_` from auth.json
- Vercel API env vars return `{created: {...}, failed: []}` not direct object
- `styled-jsx` not available in App Router — use `<style>` tags instead
- Cloud server SSH password auth was disabled — had to pivot to GitHub-based approach
- Pipeline cron running every minute can overwrite git working dir if `git add` is too broad
