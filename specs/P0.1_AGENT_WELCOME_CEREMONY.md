# Phase 0.1: Agent Welcome Ceremony — Implementation Brief

**Agent:** Forge (build execution)  
**Model:** Qwen 3.6 Plus  
**Priority:** Highest — Phase 0, first task  
**Estimated effort:** 3-4 hours

---

## What We're Building

After the WelcomeOverlay ("Hey Don! I'm Conflux...") and before the desktop renders, insert an **Agent Welcome Ceremony** — a full-screen, multi-step sequence where each selected agent introduces itself with a *personalized, pre-built starting point*.

This transforms "empty app cold-start" into "your team is already set up."

---

## User Flow

```
Onboarding → WelcomeOverlay (existing) → AgentIntroductions (NEW) → Desktop
```

The ceremony:
1. Shows one agent at a time (3-5 agents based on user selection in onboarding)
2. Each agent has a themed intro screen matching their app's visual identity
3. Shows what they've *already prepared* — not empty, not waiting
4. User clicks "Next" to progress through agents
5. Final screen: "Your team is ready. Welcome home." → drops to desktop

- Pre-built starter: Shows 3 example dream templates (Learn a Skill, Get Fit, Start a Business)

### Orbit (Life Autopilot)
> "I'm Orbit — I keep your life on track. I've set up a focus system for you. What's been keeping you up at night lately?"

- Pre-built starter: Empty focus list with 3 suggested focus areas (Health, Work, Relationships)

### Conflux (Main Agent)
> "I'm Conflux — your strategic partner. My job is to make sure all your agents work together. You're all set — let's get started."

- Pre-built starter: Overview of all connected agents, system status

## Technical Spec

### New Components

#### `src/components/AgentIntroductions.tsx`

```tsx
interface AgentIntroductionsProps {
  userName: string;
  selectedAgentIds: string[];
  familyMemberId?: string;
  onComplete: () => void;
}
```

**Behavior:**
- Full-screen overlay with animated transitions between agents
- Each agent card has themed colors matching their app's design identity
- Progress dots at the bottom
- Skip button (top right) — completes all remaining agents silently
- On completion, calls `onComplete()` which transitions to desktop (App.tsx gate)

**Agent sequence:** Always ends with Conflux last (strategic partner wraps up). Order: Hearth → Pulse → Horizon → Orbit → Conflux (only show agents the user selected).

**Animation:** Slide-in from right, hold 500ms, slide-out to left. Next agent slides in. Total ceremony: ~30-60 seconds for 5 agents, ~10 seconds if skipped.

**Data persistence:** Store `conflux-introductions-complete=true` in localStorage. Don't show again unless user resets via Settings.

### New Seed Data

#### `src/data/agent-intro-data.ts`

Contains:
- Each agent's intro copy, colors, emoji, app name
- Pre-built starter configurations (default categories, pantry items, templates)
- Input field label + placeholder per agent

### Backend Integration

#### Rust: Seed Starter Data

When the ceremony completes, the user's inputs are saved to the actual app databases:

- Kitchen pantry items → `kitchen_pantry` table
- Budget categories → `budget_categories` table  
- Dreams → `dreams` table with auto-created milestones
- Life focus items → `orbit_focus` table

**Approach:** Each agent's "starter" is created via existing Tauri commands. The ceremony just calls them on completion:
```typescript
// After all intros complete
for (const input of userInputData) {
  if (input.type === 'kitchen') {
    await invoke('kitchen_add_pantry_item', { req: input.data });
  }
  if (input.type === 'budget') {
    await invoke('budget_set_monthly_income', { userId, amount: input.data.income });
  }
  if (input.type === 'dreams') {
    await invoke('dream_add', { userId, title: input.data.goal, description: input.data.notes });
  }
  // etc.
}
```
